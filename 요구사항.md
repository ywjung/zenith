# GitLab CE 기반 ITSM Docker 통합 구성 문서

## 1. 개요

본 문서는 GitLab Community Edition(Docker) 기반으로 일반직원이 사용하기
쉬운 ITSM 포털을 구축하고, 모든 구성요소를 Docker Compose로 통합
구동하는 방법을 정의한다.

구성 요소:

-   GitLab CE (이슈 저장소 / 협업 / 감사 로그)
-   ITSM Portal Frontend (Next.js)
-   ITSM Portal Backend (FastAPI)
-   PostgreSQL (포털 설정/평가 저장)
-   Redis (세션/캐시)
-   Nginx (리버스 프록시)

------------------------------------------------------------------------

## 2. 전체 아키텍처

일반직원 → ITSM Portal → GitLab Issue 생성\
IT팀 → GitLab에서 이슈 처리\
완료 → Portal에서 완료확인 및 만족도 평가

GitLab은 Single Source of Truth 역할을 수행한다.

## 4. docker-compose.yml 예시

``` yaml
services:

  gitlab:
    image: gitlab/gitlab-ce:latest
    container_name: gitlab
    restart: unless-stopped
    shm_size: "256m"
    ports:
      - "8929:80"
      - "2224:22"
    volumes:
      - gitlab_config:/etc/gitlab
      - gitlab_logs:/var/log/gitlab
      - gitlab_data:/var/opt/gitlab

  postgres:
    image: postgres:16
    environment:
      POSTGRES_DB: itsm
      POSTGRES_USER: itsm
      POSTGRES_PASSWORD: change_me
    volumes:
      - itsm_pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7

  itsm-api:
    build: ./itsm-api
    environment:
      DATABASE_URL: postgresql+psycopg://itsm:change_me@postgres:5432/itsm
      GITLAB_API_URL: http://gitlab:80
      GITLAB_PROJECT_ID: 123
      GITLAB_ADMIN_TOKEN: glpat_xxxxx
    depends_on:
      - postgres
      - gitlab

  itsm-web:
    build: ./itsm-web
    environment:
      NEXT_PUBLIC_API_BASE_URL: http://localhost/api
    depends_on:
      - itsm-api

  nginx:
    image: nginx:latest
    ports:
      - "80:80"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
    depends_on:
      - itsm-web
      - itsm-api

volumes:
  gitlab_config:
  gitlab_logs:
  gitlab_data:
  itsm_pgdata:
```

## 5. FastAPI Dockerfile 예시

``` dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

------------------------------------------------------------------------

## 6. Next.js Dockerfile 예시

``` dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

FROM node:20-alpine
WORKDIR /app
COPY --from=build /app ./
EXPOSE 3000
CMD ["npm", "run", "start"]
```

## 7. Nginx 설정 예시

``` nginx
server {
  listen 80;

  location / {
    proxy_pass http://itsm-web:3000;
  }

  location /api/ {
    proxy_pass http://itsm-api:8000/;
  }
}
```

## 8. 초기 구동 절차

1.  docker compose up -d --build
2.  GitLab 초기 비밀번호 설정
3.  ITSM 프로젝트 생성
4.  라벨 생성 (status::, cat::, prio::)
5.  서비스 계정 토큰 생성 후 .env 반영
6.  itsm-api 재기동

------------------------------------------------------------------------

## 9. 권장 운영 사양

-   GitLab 최소 4\~8GB RAM
-   정기 백업 (GitLab Volume + Postgres Volume)
-   HTTPS 적용 권장
-   서비스 계정 토큰 최소 권한 원칙 적용
